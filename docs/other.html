Reproducible, scalable, and shareable analysis pipelines with bioinformatics workflow managers

  • Topic: A Perspective piece highlighting the necessity and benefits of using bioinformatics workflow managers to create reproducible, scalable, and shareable computational analysis pipelines for high-throughput data.
  • Function: Workflow managers simplify pipeline development, optimize resource usage, and manage software installation/versions for portability across different compute platforms.
  • Resource: The paper outlines community-curated pipeline initiatives that allow users to perform complex, best-practice analyses without manual assembly, contributing to better reproducibility in biomedical research.

Good enough practices in scientific computing

  • Core Principle: The paper outlines a set of pragmatic “good enough” practices for research computing to improve reproducibility, integrity, and efficiency in computational science, addressing the widespread lack of formal training in these areas.
  • Data Management: Recommendations include never modifying raw data, organizing files descriptively, and using version control for all processing scripts to maintain a clear data provenance trail.
  • Code and Project Quality: Researchers should write clear, human-readable code, automate their analysis workflows using tools like Makefiles, and clearly license their data and code to facilitate reuse and collaboration.

Best Practices for Scientific Computing

  • The paper outlines 24 best practices across eight core areas for scientific computing, arguing that software is an “experimental apparatus” that must be built and checked with rigor, especially since most scientific programmers are self-taught.
  • Core principles include writing human-readable programs (consistent naming, simple functions), automating repetitive tasks (using scripts and build tools), and employing incremental development with version control (VCS) for all manually created files.
  • Key recommendations for code quality and reliability involve planning for mistakes through the use of assertions and automated testing, optimizing performance only after the code works correctly and often by using a higher-level language, and documenting design/interfaces rather than implementation mechanics.

Biobanking with genetics shapes precision medicine and global health

  • Core Function: Biobanks are large-scale, longitudinal cohorts linking genomic data and electronic health records (EHRs) to enable high-powered analyses for precision medicine, including GWAS, Polygenic Risk Scores (PRS), and Pharmacogenomics (PGx).
  • Key Applications: Biobanks facilitate the discovery of novel genetic loci for complex diseases, improve the predictive performance of PRS for conditions like multiple sclerosis, and highlight the necessity of population-specific PGx screening.
  • Major Challenge: The field’s greatest limitation is the severe underrepresentation of non-European ancestries, which results in the poor transferability and reduced utility of genetic risk models across diverse global populations.

simona: a comprehensive R package for semantic similarity analysis on bio-ontologies

  • Goal: Introduces simona, an R package providing a comprehensive and efficient solution for semantic similarity analysis on general bio-ontologies.
  • Features: The package is highly flexible, implementing over 70 diverse semantic similarity methods, utilizing efficient data structures and fast ontology traversal.
  • Result: A benchmark confirmed that methods cluster by mathematical approach, and that similarities based on ontology topology alone are highly effective for revealing semantic relationships.
